VS Code, Gitlab, and AWS: Upload swagger files to devdocs repository

Once you've created the swagger files and converted them from YAML to JSON, you'll need to upload them to the Gitlab devdocs repository. This is where the code for the developer documentation lives before its pushed to the AWS bucket.

Locating WSL files in Windows Explorer

In order to upload the swagger files from your local instance of the redrock-swagger repository to the remote instance of the devdocs repository, those files must be located on your local machine.

The general process for locating files in WSL is described in this article:

Locate the converted JSON files

The specific files that need to be moved from the redrock-swagger repository to the devdocs repository are the most recently converted JSON files.

If these files were installed to the root directory of your Ubuntu instance, by default they will be located at:

\\wsl$\Ubuntu\home\USERNAME\redrock-swagger\dist\swagger\vXX.X

With the exception that this file path requires:

  • No spaces
  • USERNAME is the username that corresponds to your personal instance of Ubuntu
  • vXX.X corresponds to the API version number which was recently updated and converted to JSON files

Upload files to the devdocs repository:

At the time that this documentation was last edited, all technical writers have the redrock-swagger repository installed to WSL because this is a requirement for it to function. If you've used this documentation to clone the devdocs repository locally, then your devdocs repository will also be located in WSL and accessible from Ubuntu.

However, at the time that this documentation was last edited, there are some early adopters of this process that are running the local redrock-swagger repository from WSL and the local devdocs repository from Windows. Therefore, this section will be divided into two separate parts:

  1. Instructions for technical-writers that have both the redrock-swagger and devdocs repositories cloned to WSL
    1. From Windows Explorer copy all of the JSON files contained in the subdirectory \redrock-swagger\dist\swagger\vXX.X of Ubuntu.
    1. Without leaving the Ubuntu directory in Windows Explorer, navigate to the local instance of the devdocs repository, which by default should be located at:

      \\wsl$\Ubuntu\home\USERNAME\devdocs\

    1. Continue navigating through Windows Explorer, and open the file folder located at:

      \devdocs\src\res\swagger\

    1. Following the naming conventions of the other folders, create a new folder called swaggerXXXXXX where XXXX is of the date format MMDDYY and is equal to the same date that was added to the recently converted JSON files. In the above image, that date would be 042221
    1. After the new folder is created, navigate into the folder \devdocs\src\res\swagger\swaggerXXXXXX\ and then paste the JSON files—originally copied from the redrock-swagger repository—into the folder.
    1. Open the devdocs repository in VS Code and select the Save All option from the File menu

    The devdocs repository should now house its own copy of the updated swagger files. However, those files have not been updated in the HTML for the website and will not yet be accessible by the user.

    Before proceeding to the next article to update the HTML code for the website, add, commit, pull, and then push your changes to the devdocs repository.
