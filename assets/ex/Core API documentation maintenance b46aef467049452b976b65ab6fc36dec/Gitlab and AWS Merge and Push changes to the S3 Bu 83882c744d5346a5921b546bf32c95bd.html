Gitlab and AWS: Merge and Push changes to the S3 Bucket

The code for developer.workfront lives in the devdocs repository, but the actual website is hosted from an AWS S3 Bucket. In order to update the live site, the code must be pushed from the devdocs repository to the S3 Bucket.

Merge changes up from feature to Master

Before changes can be pushed from the devdocs repository to the AWS S3 Bucket, the changes made to feature need to be merged up to the master branch of devdocs. In order to preserve data integrity, this process should only be performed by the primary repository owner.

The steps to merge the feature branch into the master branch of the devdocs repository are identical to the steps for merging the feature branch into the master branch of the redrock-swagger repository. A detailed description of that process can be found here:

Push changes from the master branch to AWS

To push changes from the master branch to the corresponding AWS Bucket:

  1. From the Repository Overview navigate to the Main Menu and hover over CI/CD, when the CI/CD options appear, select Pipelines
  1. From the pipelines overview, navigate to the blue button labeled "Run pipeline", located in the upper-right corner of the screen, and click it.

  1. From the Run pipeline page, leave all values to their default and click the blue button labeled "Run Pipeline"

  1. This initiates the CI/CD protocols that send the updated master branch contents to the AWS S3 Bucket, once the load indicators are finished, the updated code has been pushed to the corresponding AWS S3 bucket.
